# Plan 09-03: Add Interactive Review Queue

## Requirements

- REV-03: User can run a review queue that replays samples one by one and
  accepts category, speaker, and answer-decision corrections from one command
  flow.

## Tasks

- Add `trigger-samples review` with the same filters as list.
- For each selected sample, print stored/current classification and optionally
  replay audio.
- Accept category, speaker, decision, skip, and quit actions.
- Support `--limit` and `--no-play` for deterministic testing.

## Verification

- CLI tests using `input=` to correct category/speaker/decision and to skip.
