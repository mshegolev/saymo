# Plan 08-01: Add Session Metadata And Storage Helpers

## Requirements

- SES-01: User can start a named trigger-capture session and have every saved
  sample record the session id in its JSON metadata.
- SES-02: User can finish a session and see persisted summary counts.
- SES-03: User can list prior capture sessions and inspect training-readiness
  status.

## Tasks

- Add trigger-session helper dataclasses for session metadata, summaries, and
  readiness status.
- Add helper functions to create session ids, locate ledger files, write
  running/finished ledgers, and summarize sample records.
- Extend `save_trigger_sample` with optional session fields while preserving
  backwards compatibility for old callers.
- Add unit tests for metadata persistence, summary counts, and readiness labels.
