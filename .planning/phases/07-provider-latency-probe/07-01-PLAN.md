# Plan 07-01: Add Provider Latency Probe Command

## Requirements

- LAT-01: User can run a provider-specific latency probe against an active
  Chrome call using the existing provider abstraction.

## Tasks

- Add `saymo provider-latency`.
- Resolve provider from `meetings.<profile>.provider` with an override flag.
- Support `--text` for deterministic dry runs and call-audio capture when text
  is omitted.
- Check provider tab readiness and BlackHole output before playback.

## Verification

- CLI tests for a successful provider probe with mocked provider and playback.
- CLI tests for a blocked provider-tab path.
