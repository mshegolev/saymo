# 13-02 Plan: Add Speaker Suggestion Apply/Reject Metadata

## Objective

Let users accept, reject, or override one suggestion without losing the original
speaker id or suggested label.

## Tasks

1. Extend suggestion sidecar records with review status, reviewed label, and
   reviewed timestamp fields.
2. Add pure helpers for finding and reviewing a suggestion by sample path or
   session sequence.
3. Add `trigger-samples speaker-suggestion` actions for show, accept, reject,
   and override.
4. Wire prefixed review-queue actions for accepting/rejecting/overriding the
   displayed suggestion.

## Verification

- Tests prove accept/override updates sample JSON `speaker`, reject does not,
  and the sidecar still preserves original suggestion fields.
