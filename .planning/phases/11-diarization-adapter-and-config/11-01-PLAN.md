# Plan 11-01: Diarization Config And Availability Checks

## Goal

Add disabled-by-default diarization configuration and backend availability
helpers without importing optional ML packages during normal Saymo startup.

## Requirements

- DIAR-01
- DIAR-02
- DIAR-03

## Tasks

- Add `DiarizationConfig` to `saymo.config.SaymoConfig`.
- Add env-safe fields for engine, model id, device, min/max speakers, and token
  env name.
- Add analysis helpers that normalize config and report disabled/missing/ready
  availability states.
- Cover default-disabled, configured pyannote, missing token, and env token
  paths with tests.

## Verification

- `.venv/bin/python -m pytest -q tests/test_config.py tests/analysis/test_diarization.py`
