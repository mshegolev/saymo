# Milestone Audit: v1.3 Local Diarization Assist

## Verdict

Status: passed

All v1.3 requirements are satisfied. The milestone keeps diarization optional,
stores inferred speaker identity in local sidecars, requires explicit review
before promotion into training labels, and reports speaker-label quality without
exporting raw audio or transcript text.

## E2E User Flows

1. User checks optional backend availability with `saymo diarization-check`.
2. User diarizes a completed capture session with `trigger-sessions diarize`.
3. User maps diarization clusters with `trigger-sessions map-speaker`.
4. User reviews one sample via `trigger-samples speaker-suggestion --accept`,
   `--reject`, or `--label`.
5. User exports speaker quality with `trigger-sessions speaker-report`.
6. User runs `trigger-eval`, `trigger-classifier readiness`, or training with
   manual sample labels remaining authoritative.
