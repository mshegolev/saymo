# Plan 08-02: Wire Session IDs Into Trigger Capture

## Requirements

- SES-01: `trigger-capture` accepts a named session and writes the session id
  into all saved sample metadata.
- SES-02: `trigger-capture` prints a final summary including saved samples,
  skipped silence windows, categories, speakers, answer decisions, and ledger
  path.

## Tasks

- Add `--session` to `saymo trigger-capture`.
- Create the session ledger before audio capture starts.
- Track skipped silence windows when `--save-silence` is not enabled.
- Always finish the ledger in a `finally` path for normal duration expiry and
  Ctrl+C stops.
- Include the session id in capture output so the user can inspect it later.

## Verification

- CLI tests for the `trigger-capture --help` option.
- Focused tests for final summary rendering using mocked sample records.
