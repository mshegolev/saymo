# Plan 05-01: Add Speaker Label Metadata Support

## Requirements

- SPK-01: User can attach and read local speaker labels for captured trigger
  samples from local metadata without requiring cloud diarization.

## Tasks

- Add `speaker` metadata to newly saved trigger samples.
- Load legacy samples without `speaker` as `unknown`.
- Show speaker labels in sample list/replay/report output.

## Verification

- Unit/CLI tests for new sample metadata and legacy default behavior.
