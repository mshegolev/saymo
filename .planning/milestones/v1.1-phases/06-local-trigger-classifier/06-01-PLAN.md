# Plan 06-01: Add Accepted/Rejected Sample Labels

## Requirements

- CLF-01: User can mark a sample's answer decision as accepted or rejected from
  the CLI without editing JSON manually.

## Tasks

- Load legacy samples without `answer_decision` as `unlabeled`.
- Add `trigger-samples decision SAMPLE.json --decision accepted|rejected|unlabeled`.
- Show the answer-decision label in sample list, replay, and sanitized report
  output.

## Verification

- CLI tests for writing the decision label and reading it back through
  list/replay/report flows.
